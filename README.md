# .github
## Описание проекта
Проект metering-control-system разрабатывается в рамках учебного проекта. В планах разработать приложение для монтеров, которое позволит фиксировать результат работы и платформу, которая позволит контролировать выполнение работы по монтажу ПУ.

## Правила публикации в репозитории
### Создание веток
❗️При создании ветки она должна иметь название вида `<тип ветки>/<название ветки>`
Возможные типы веток:
- feature/ – новые фичи
- bugfix/ – исправление багов
- refactor/ – рефакторинг (улучшения существующего кода, без изменения его функциональности)
- chore/ – вспомогательные задачи (настройка CI/CD, конфиги)
- sandbox/ – тестовый код, который не будет сливаться с develop
### Оформление коммитов
❗️При создании коммита он должен иметь такое оформление `<тип коммита>: <описание коммита>`
❗️Описание коммита подчиняется правилам:
- Все коммиты на английском.
- Не используйте большие буквы в заголовках.
- Используйте настоящий глагол в настоящем времени. Вместо “added feature” или “fixed bug” используйте “add feature” или “fix bug”.
- Придерживайтесь описания в пределах 50–72 символов. Заголовок должен быть понятен без необходимости читать дополнительную информацию.

❗️Типы коммитов:
- feat: добавление новой функциональности.
- fix: исправление ошибок.
- style: правки, касающиеся стиля (например, отступы, форматирование), без изменения логики.
- refactor: изменения в коде, не добавляющие новой функциональности и не исправляющие ошибок, но улучшающие структуру.
- chore: изменения, не относящиеся к коду (например, обновление зависимостей или настройка CI/CD).
